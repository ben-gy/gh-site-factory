## Idea Source
IDEAS.md (quote): "Fully interactive Australian crime data explorer by LGA/region, standardised across all states. States currently report crime data differently (inconsistent time periods, terminology, geographic resolutions). Site ingests and normalises all state datasets into a single schema so users can explore, filter, and compare crime stats state-by-state and LGA-by-LGA in a consistent interface."

## Architecture Decisions
- **Vanilla TypeScript** over React: the UI is view-based with minimal shared state, so Vanilla keeps the bundle small (~73 KB). All views are implemented as `renderXxx(container)` functions that rebuild their DOM on route change.
- **Hand-rolled SVG** for all charts (bars, trend lines, matrix heatmap). No chart libraries needed — saves ~150 KB of dependencies. Each chart is ~50-100 LOC of focused code.
- **Leaflet** kept as the only external viz dependency, for the map view. Loaded from CDN at runtime rather than bundled, so initial page load isn't affected.
- **Embedded data with pipeline for updates**: the initial data is in `src/data.ts` using rates-per-100K profiles derived from publicly published ABS Recorded Crime statistics. A GitHub Actions pipeline refreshes state detail weekly via QLD/SA CKAN APIs and NSW BOCSAR. Embedded data is type-safe; pipeline-updated data lives in `public/data/pipeline-data.json` for future use.
- **Hash-based routing** (not History API) for GitHub Pages compatibility without server rewrites.
- **Light theme** for the civic/consumer audience — dark terminal aesthetic would feel wrong for a public data tool.
