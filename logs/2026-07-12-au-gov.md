## Architecture Decisions

- **Vanilla TypeScript, no framework.** The app has 9 views but shares 3 utilities (glossary tooltip, detail panel, about modal) — a component tree was not needed. Bundle: 217 KB / 65 KB gzipped.
- **Pipeline > runtime-fetch.** AGOR only updates quarterly, so a weekly GitHub Actions job that fetches the latest CSV, normalises it to compact JSON, and commits to `public/data/` keeps the site fast and free of data.gov.au CORS.
- **No chart libraries.** Treemap uses a squarified layout in ~60 lines. Timeline uses hand-rolled stacked bars. Network uses a hand-rolled force simulation (~50 lines). Only the map view uses a real library (Leaflet).
- **Simplified AU state GeoJSON bundled locally.** No third-party fetch needed. 1.87 KB.
- **Curated ministers list rather than another data source.** Ministerial data isn't in AGOR; hand-curating 32 ministers to match the current ministry list was faster and more accurate than trying to scrape parliamentary websites at runtime.

## Views implemented

1. **Portfolio tree** — collapsible hierarchy, one card per portfolio; entities grouped by classification (A/B/C/D). Departments of State are pinned to the top of A. Primary body groups.
2. **Cabinet & Ministry** — Cabinet ministers (gold border), outer ministry (portfolio-coloured), then a portfolio-to-Department-of-State grid.
3. **Treemap** — squarified layout, sized by materiality-weighted counts or raw counts, portfolio-coloured, click-to-open cells.
4. **Portfolio × Body type matrix** — heat map with 17 portfolios × 13 body types; click a cell → filters the Table.
5. **Relationship graph** — hand-rolled force simulation, 1,381 nodes / 775 parent-child edges. Filterable by portfolio; can filter to Material bodies only.
6. **Map** — Leaflet + local AU state GeoJSON, head office locations clustered by suburb, popups with portfolio breakdown per cluster.
7. **Timeline** — stacked bars by decade (default) or by year (last 25). Portfolio-coloured stacks; click segment → filters Table by portfolio + year range.
8. **Searchable table** — full-text search across name/description/act/parent/location; sort by any column; type filter; material-only checkbox; drill-down detail panel.
9. **Insights** — auto-computed cards (entity count, material count, oldest, most recent, largest portfolio, most common body type, Canberra dominance, new bodies in last 3 years, joint ventures & interests) + materiality leaderboard.

## Errors & Resolutions

- **RequestAnimationFrame throttled in 0×0 headless preview** — the network view's initial `simulate()` scheduled all work via `requestAnimationFrame` which never fired in the preview environment (the browser had a 0×0 viewport). Refactored `simulate()` to build the graph and run 30 initial tick iterations synchronously, then continue with RAF for smooth animation. Works identically in normal browsers, and works in the preview.
- **DNS + TLS cert cycle** — used the standard cycle trick (set CNAME → clear → re-set) to trigger Let's Encrypt issuance.
