## Architecture Decisions

- **Vanilla TS over React.** Single-page app with six tab views; no nested state tree, no routing library. Vanilla TS keeps bundle small (206 KB JS / 62 KB gzipped).
- **Runtime fetch over pipeline.** The feed refreshes every 5 s and is CORS-open, so a GitHub Actions pipeline would only add latency. Polling every 10 s with `AbortController` and `visibilitychange` pause-when-hidden keeps bandwidth polite.
- **Leaflet for map, hand-rolled SVG for everything else.** The flow diagram (Sankey-style), dish × spacecraft matrix, per-dish compass, and session timeline are all hand-rolled SVG. Avoids pulling in D3 or a chart library.
- **Curated spacecraft directory.** `src/data/spacecraft.ts` hand-maintains friendly names, mission descriptions, agency, launch year, and category for ~40 missions the DSN regularly tracks. Unknown codes fall back to a placeholder that preserves the raw identifier.
- **Glossary module + click-to-show tooltips.** Follows the Artemis Tracker pattern — `.glossary-link` spans with `data-term` attributes, a fixed-position tooltip, click to show, escape or outside-click to dismiss.
- **Dark "planetarium" aesthetic** — deep indigo/black background with warm amber and cool cyan accents. Chosen over the generic dark-hacker look because the audience is aspirational (space wonder), not technical monitoring.

## Errors & Resolutions

1. **TypeScript errors on first `npm run build`**
   - `unused import formatPower` in `views/now.ts` → removed
   - `Record<Band, string>` index signature with `string` key → cast `p.bands[0] ?? ''` to `Band` in both `flow.ts` and `matrix.ts`
2. **Session dedup test failure**
   - `updateSession` only compared against the last history entry; a single snapshot with two pairs would cause false dedup on the second pass. Fixed by searching the history for the last matching `(dish, spacecraft)` pair and comparing its timestamp to the new snapshot's timestamp. Test now passes.
3. **preview_start harness error**
   - `spawn /Applications/Claude.app/Contents/Helpers/disclaimer ENOENT` when trying to use the Launch Preview panel — an environment-specific issue unrelated to the build. Fell back to `npm run preview` in background and verified via Chrome MCP + curl. Site rendered correctly with 3 complexes and 14 dish cards, live feed updated.
4. **TLS cert issuance**
   - `https_enforced` sat at `false` for the first 5 polling attempts. Re-cycling the CNAME (empty then re-set) flipped it to `true` on the next check.
