# Build Log: Income by Postcode
**Date:** 2026-07-18
**Status:** deployed

## Idea Source
Researched. IDEAS.md was empty. Scanned the registry (39 sites) and `gh repo list ben-gy`
for a genuine gap: nothing in the catalog covered **income, tax, or what people earn** —
despite it being one of the most-Googled civic-data questions in Australia. Confirmed the
ATO publishes postcode-level taxation statistics on data.gov.au, verified the current
edition (2023-24) is live with Tables 6 (161 items × postcode), 7 (income/age/occupation
bands) and 8 (median + average income across 12 year-points), and validated the central
thesis in the raw data before committing:

- Toorak (3142): median $82,873, average $277,708 → 3.35× gap (a rich tail).
- Fortescue/Paraburdoo/Rocklea (Pilbara): the highest *medians* in Australia
  ($142,580 / $134,252 / $111,679) with averages at or below the median (~0.90–1.03×).

The typical Toorak earner makes less than a Karratha mine worker — a real, non-obvious
finding that falls straight out of the data. Distinct data shape from au-jobs (LGA
unemployment) and au-solar (postcode installs), so a new site, not an expansion.

## Site Details
- **Name:** Income by Postcode
- **Repo:** ben-gy/au-income
- **Category:** economy (data-explorers in the public index)
- **Audience:** Australians Googling "average income in my suburb"; homebuyers, renters,
  journalists, policy/economics people.
- **Stack:** Vanilla TypeScript + Vite 6 + Leaflet 1.9; Vitest.
- **Data strategy:** pipeline. Cron is **yearly** (`23 6 9 6 *`, 9 June) — the ATO
  publishes taxation statistics once a year, so a yearly cron matches the source exactly.
  Monthly/weekly would just re-download an identical workbook.

## Data Sources
- ATO Taxation Statistics 2023-24 — Individuals Table 6/7/8 (data.gov.au dataset
  `taxation-statistics-2023-24`)
- ABS ASGS 2021 Postal Area digital boundaries (POA_2021_AUST_GDA2020_SHP.zip, 56 MB →
  mapshaper 1.2% → 1.9 MB GeoJSON, 2,266 polygons matched to data)
- matthewproctor/australianpostcodes (postcode → locality names)

## Architecture Decisions
- **Vanilla TS over React:** tab-switched single page, no routing tree or cross-component
  state — React would be dead weight.
- **exceljs (streaming) to parse the 8 MB workbooks** in Node; ~1.1s per file, preserves
  numeric types. Columns matched by header *text*, never index (the ATO reorders columns
  between editions; a silent index shift would publish wrong numbers instead of failing).
- **Log-log scatter for The Gap:** incomes span ~$20k–$320k and cluster at the low end; a
  linear axis squashes 90% of the country into a corner.
- **Quantile choropleth:** income is so skewed that equal-interval breaks paint the whole
  country one colour and only Toorak stands out.
- Copied canonical patterns (tooltip, svgZoom, leafletMap) rather than re-rolling.

## Data Traps Handled (documented in code + tests)
1. `na` = privacy-suppressed, coerced to `null` NOT `0` (0 would drag every average/rate down).
2. Table 7A carries BOTH `$180,000+` and a `$120,001+` rollup; summing double-counts every
   high-income postcode. Rollup used only as a fallback (232 postcodes need it).
3. The postcode reference CSV quotes every value; a naive split left the quotes attached
   (`"3142"`) and unnamed every suburb. Reused the RFC4180 parser from au-solar.
4. exceljs returns footnoted header cells ("Median3 …") as rich-text objects; `String()`
   yields "[object Object]" and every column lookup fails — unwrapped in normHeader.
5. ATO reports rental losses as negative dollars; stored magnitude so "avg loss" reads +ve.

## Bugs Found & Fixed During Verification
- **Drill-down distribution bars invisible** — `.bf` fill is a `<span>` (inline), so
  width/height were ignored and every bar rendered 0×0. A screenshot alone would have
  passed it; caught by asserting `getBoundingClientRect().width` in the browser. Fixed with
  `display:block`.
- **Trend end-labels collided** into one unreadable pile; added a declutter pass with
  leader lines.
- **CI deploy failed at `npm test`** — tests/pipeline.test.ts imported xlsx.mjs, which
  imports exceljs (a pipeline-only dep the root `npm ci` never installs). Split the
  dependency-free parsing primitives into pipeline/lib/parse.mjs; verified tests pass with
  pipeline/node_modules removed (CI conditions).
