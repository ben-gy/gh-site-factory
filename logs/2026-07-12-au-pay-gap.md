## Data Sources
- WGEA Employer Gender Pay Gaps spreadsheet — https://www.wgea.gov.au/sites/default/files/documents/Employer-Gender-Pay-Gaps-Spreadsheet.xlsx (8,616 employers, 1,837 corporate groups; 2024-25 reporting year published March 2026, 2023-24 alongside)
- WGEA Data Explorer / methodology — https://www.wgea.gov.au/Data-Explorer (glossary + interpretation)

Note: the raw WGEA public data file (952 MB) contains workforce composition but no remuneration, so per-employer pay gaps cannot be computed from it — the published spreadsheet (the figures WGEA itself calculates) is the correct source and was used.

## Architecture Decisions
- **Vanilla TS over React:** single-page, multi-view dashboard with hand-rolled views and a small central store — no routing library or heavy component tree needed. Smaller bundle (48 KB JS gzip 16.5 KB).
- **Pipeline over embedded/runtime-fetch:** the source is a 2 MB xlsx that needs parsing; committing pre-parsed JSON gives instant load, and a weekly GitHub Action refreshes it when WGEA republishes (annually).
- **No chart libraries:** every visualisation (diverging industry bars, quartile pyramids, histogram, heatmap, YoY movers, correlation scatter) is hand-rolled SVG.
- **No map:** the data has no geography (it's by ABN/industry/sector/size), so per the rules a map was intentionally omitted.
- **Default sort = signed gap descending:** the source genuinely contains extreme median-GPG outliers (down to −501% at tiny/skewed employers, a real WGEA artifact of median maths when one gender is scarce in a quartile). Sorting by gap size surfaced these first and read as broken data; sorting by signed gap leads with the largest male-favouring gaps (the expected narrative, ≤88%) while keeping outliers accessible via sort/search. Data is never capped — all 8,616 employers are included.
